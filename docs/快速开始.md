# 快速开始指南

## 安装

### 方式一：从 VSCode 市场安装（推荐）

1. 打开 VSCode/Cursor
2. 点击左侧扩展图标（或按 `Ctrl+Shift+X`）
3. 搜索 "HT Cursor Rules"
4. 点击 "安装"

### 方式二：从 VSIX 文件安装

1. 下载最新的 `.vsix` 文件
2. 打开 VSCode/Cursor
3. 按 `Ctrl+Shift+P` 打开命令面板
4. 输入 "Extensions: Install from VSIX..."
5. 选择下载的 `.vsix` 文件

### 方式三：本地开发安装

```bash
# 克隆仓库
git clone https://github.com/your-username/ht-cursor-rules.git

# 进入目录
cd ht-cursor-rules

# 安装依赖
npm install

# 编译
npm run compile

# 打包
npm run package

# 安装生成的 .vsix 文件
```

## 第一次使用

### 1. 打开项目

在 VSCode/Cursor 中打开你的项目文件夹。

### 2. 添加规则

#### 方法 A：使用命令面板（推荐）

1. 按 `Ctrl+Shift+P`（Mac: `Cmd+Shift+P`）
2. 输入 "HT"
3. 选择 "HT: 添加 Cursor 规则文件"
4. 从列表中选择规则类型
5. 确认操作

![命令面板](../assets/command-palette.png)

#### 方法 B：使用右键菜单

1. 在资源管理器中右键点击任意文件夹
2. 选择 "HT: 添加 Cursor 规则文件"
3. 选择规则类型并确认

### 3. 选择规则类型

插件提供以下规则类型：

| 类别 | 规则名称 | 适用场景 |
|------|---------|---------|
| **本地开发** | Python | Python 项目开发 |
| **网站开发** | HTML/CSS/JavaScript | 传统网站开发 |
| | React | React 项目开发 |
| | Vue.js | Vue 项目开发 |
| **移动开发** | iOS 应用开发 | iOS 原生应用 |
| | Android 应用开发 | Android 原生应用 |
| **小程序开发** | 微信小程序 | 微信小程序项目 |
| **浏览器插件** | Chrome 插件 | 浏览器扩展开发 |
| **通用规则** | 通用开发规则 | 所有项目 |

### 4. 处理已有规则

如果项目中已经存在 `.cursorrules` 文件，你可以选择：

- **合并**：将新规则添加到现有规则后面（推荐）
- **覆盖**：用新规则替换现有规则
- **取消**：放弃操作

### 5. 查看规则文件

添加成功后，点击 "查看文件" 可以立即查看和编辑规则内容。

规则文件位置：`项目根目录/.cursorrules`

## 使用示例

### 示例 1：React 项目

```bash
# 1. 创建 React 项目
npx create-react-app my-app
cd my-app

# 2. 在 VSCode/Cursor 中打开项目

# 3. 添加 React 规则
# 按 Ctrl+Shift+P，选择 "HT: 添加 Cursor 规则文件"
# 选择 "React"

# 4. 开始使用 Cursor AI 进行开发
# Cursor 会根据规则文件提供更准确的代码建议
```

### 示例 2：Python 项目

```bash
# 1. 创建 Python 项目
mkdir my-python-project
cd my-python-project
python -m venv venv

# 2. 在 VSCode/Cursor 中打开项目

# 3. 添加 Python 规则
# 按 Ctrl+Shift+P，选择 "HT: 添加 Cursor 规则文件"
# 选择 "Python"

# 4. 开始使用 Cursor AI 进行开发
```

### 示例 3：添加多个规则

你可以为一个项目添加多个规则，例如：

1. 添加 "通用开发规则"（基础规范）
2. 再添加 "React"（React 特定规范）
3. 选择 "合并" 选项

最终的 `.cursorrules` 文件会包含两个规则的内容。

## 自定义规则

添加规则后，你可以：

1. 打开 `.cursorrules` 文件
2. 根据项目需求修改规则
3. 添加自定义规则

示例：

```markdown
# 项目特定规则

## 命名规范
- 组件文件使用 PascalCase
- 工具函数使用 camelCase
- 常量使用 UPPER_SNAKE_CASE

## 文件组织
src/
  components/    # React 组件
  hooks/         # 自定义 Hooks
  utils/         # 工具函数
  services/      # API 服务
  types/         # TypeScript 类型
```

## 更新规则

要更新已添加的规则：

1. 再次运行添加命令
2. 选择相同的规则类型
3. 选择 "覆盖" 选项

## 常见问题

### Q: 规则文件不生效？

**A**: 确保：
- 文件名为 `.cursorrules`（注意前面的点）
- 文件位于项目根目录
- 重启 Cursor 编辑器

### Q: 如何删除规则？

**A**: 直接删除项目根目录的 `.cursorrules` 文件即可。

### Q: 可以自定义规则模板吗？

**A**: 当前版本暂不支持，但你可以：
- 添加规则后手动编辑
- 在未来版本中我们会添加自定义模板功能

### Q: 规则会影响性能吗？

**A**: 不会。规则文件只是提供给 AI 的上下文信息，不会影响编辑器性能。
