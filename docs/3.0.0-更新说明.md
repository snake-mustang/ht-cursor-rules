# HT Cursor Rules 3.0.0 更新说明

## 🎉 重大更新概览

HT Cursor Rules 3.0.0 版本是一次重大的架构升级，专注于提供更专业、更完整的开发规则支持，特别是新增了游戏开发类别。

## 📁 全新的文件夹结构

### 规则文件夹组织

规则文件不再存放在根目录，而是按类别分别存储在子文件夹中：

```
rules/
├── frontend/           # 前端框架（Angular, React, Vue, Next.js 等）
├── backend/            # 后端框架（FastAPI, Django, NestJS, Go 等）
├── mobile/             # 移动开发（React Native, Flutter, SwiftUI 等）
├── css-styling/        # CSS 和样式（Tailwind CSS 等）
├── state-management/   # 状态管理（Redux 等）
├── database-api/       # 数据库和 API（GraphQL 等）
├── testing/            # 测试框架（Cypress, Playwright, Jest 等）
├── hosting/            # 托管和部署（Netlify 等）
├── build-tools/        # 构建工具（Webpack, Vite 等）
├── language-specific/  # 编程语言（Python, TypeScript, C++ 等）
├── game-development/   # 游戏开发（Unity, Unreal, Cocos, LayaBox）
└── other/              # 其他规则
```

### 优势

- **更清晰的组织结构**：规则按功能分类，便于查找和管理
- **更容易扩展**：新增规则只需放入对应文件夹
- **更好的维护性**：规则文件隔离，互不影响
- **更直观的浏览**：文件夹结构反映了技术栈的实际分类

## 🚀 如何使用

### 1. 更新插件

如果你已经安装了旧版本，请更新到 3.0.0：
- 在 VSCode 扩展市场搜索 "HT Cursor Rules"
- 点击更新按钮

### 2. 使用游戏开发规则

1. 打开你的游戏项目
2. 按 `Ctrl+Shift+P` (Windows) 或 `Cmd+Shift+P` (Mac)
3. 输入 "HT" 并选择 "HT: 添加 Cursor 规则文件"
4. 选择 **🎮 Game Development** 类别
5. 选择你使用的游戏引擎：
   - Unity (C#)
   - Unreal Engine (C++)
   - Cocos Creator (TypeScript)
   - LayaBox (TypeScript)
6. 确认添加

### 3. 规则文件位置

规则会添加到项目根目录的 `.cursorrules` 文件中。

### 4. 规则管理

- **覆盖**: 替换现有规则
- **合并**: 将新规则追加到现有规则
- **取消**: 保持现有规则不变

    id: string;
    name: string;
    description: string;
    icon: string;
    folder: string; // 🆕 子文件夹名称
}

// 规则路径支持子文件夹
const templatePath = path.join(
    context.extensionPath,
    'rules',
    category.folder,  // 🆕 使用类别的文件夹
