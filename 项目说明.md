# HT Cursor Rules - 项目说明

## 项目概述

**HT Cursor Rules** 是一个专为 Cursor AI 编辑器设计的 VSCode 插件，旨在帮助开发者快速配置和管理 AI 编程助手规则。

### 核心功能

1. **多语言支持**：提供 9 种主流开发场景的预设规则
2. **智能管理**：支持规则覆盖、合并、预览等操作
3. **即装即用**：无需复杂配置，安装后立即可用
4. **灵活定制**：可在预设规则基础上自由修改

## 核心代码说明

### extension.ts

主要功能模块：

1. **activate()**: 插件激活入口
   - 注册命令
   - 初始化插件

2. **showRuleSelector()**: 显示规则选择器
   - 展示可用规则列表
   - 处理用户选择

3. **addCursorRules()**: 添加规则到项目
   - 读取规则模板
   - 检测已有规则
   - 执行覆盖或合并操作

4. **mergeRules()**: 合并规则内容
   - 智能合并新旧规则
   - 保持格式一致

## 开发指南

### 本地开发

```bash
# 1. 克隆项目
git clone https://github.com/your-username/ht-cursor-rules.git
cd ht-cursor-rules

# 2. 安装依赖
npm install

# 3. 编译
npm run compile

# 4. 调试（在 VSCode 中按 F5）
```

### 添加新规则

1. 在 `rules/` 目录创建新文件：`新规则.cursorrules`
2. 在 `src/extension.ts` 的 `RULE_TEMPLATES` 数组中添加：

```typescript
{
    id: 'new-rule',
    name: '新规则',
    description: '新规则描述',
    category: '分类名称',
    fileName: '新规则.cursorrules'
}
```

3. 重新编译并测试

### 发布流程

```bash
# 1. 更新版本号（package.json）
# 2. 更新 CHANGELOG.md
# 3. 编译和打包
npm run compile
npm run package

# 4. 生成 .vsix 文件
# 5. 发布到 VSCode Marketplace
npm run publish
```

| 特性 | HT Cursor Rules | sy-ai-commit |
|------|----------------|--------------|
| **主要功能** | 配置 Cursor AI 规则 | 自动化 Git 提交 |
| **使用场景** | 开发规范指导 | Git 工作流优化 |

### 互补使用

两个插件可以配合使用：

1. **HT Cursor Rules**：配置开发规范（Python、React、Vue等）
2. **sy-ai-commit**：配置 Git 提交规范 + 快捷键

```bash

1. 安装 HT Cursor Rules → 添加开发规则（如 React）
2. 安装 sy-ai-commit → 配置 Git 提交规则
3. 最终 .cursorrules 包含：React 开发规范 + Git 提交规范
```

## 常见问题

### 为什么需要这个插件？

Cursor AI 通过 `.cursorrules` 文件来了解项目规范，但手动编写规则费时费力。本插件提供：
- 预设的高质量规则模板
- 快速添加和管理
- 持续更新和维护

### 与其他类似工具的区别？

- **更专注**：专为 Cursor AI 设计
- **更全面**：覆盖 9 种主流开发场景
- **更易用**：即装即用，无需配置
- **更灵活**：支持覆盖、合并、自定义

### 规则会自动更新吗？

当前版本不会自动更新已添加的规则。未来版本会支持：
- 规则版本检测
- 一键更新规则
- 规则更新提醒
